linux环境hadoop开发准备工作

所有文章出现的代码，将会出现在我的github中，名字可以根据类全名来找，我在github中的文件夹也会加目录备注。

准备工作

现在我们开始在linux中对hadoop的API进行调用开发，在启动之前，把内存从512M修改成1G再启动

再把linux的启动设置成图形化界面，通过修改/etc/inittab中的最后一行，把3改成5，修改完成之后重启

安装linux版eclipse

大家可以上官网下载，如果不会安装，点这里，在上传完eclipse到linux中并且解压完毕之后，进入linux系统

上传成功之后，检查home目录，是否有指定的文件

如果有，在CRT中输入解压指令：

tar -zxvf eclipse-jee-neon-R-linux-gtk.tar.gz -C ./app

下图是动态图:

安装成功之后，返回虚拟机，开启eclipse

我们可以给eclipse建立一个快捷方式，即前面说的软连接并到桌面上：右击eclipse图标，选择make link

剪切到桌面:

剪切完毕:

为eclipse修改图标:右击eclipse图标，选择properties选项

在弹出来的窗口中，左击红色框中的图标

在弹出的窗口中，选择eclipse所在的/app/eclipse文件目录。

到了eclipse目录之后，选择eclipse的图标文件，icon.xpm，选择完毕后点击完成

设置完毕的图标：

开启hadoop

指的是开启hdfs，和yarn，此时可以不用CRT远程登陆，直接在登陆的linux中开启一个终端操作即可。

验证是否启动成功

在终端中输入hdfs dfsadmin -report 会出现一些参数，来判断是否开启成功，或者直接在网站URL中输入http://xinpan2:50070

在浏览器验证:

输入网址之后，跳转到页面，图中的红色框中的内容点进去可以看到hadoop的目录，在文件目录中输入/代表根目录

如果输入地址无法连接，在Linux中关闭掉防火墙，即指定的端口被限制访问，输入如下指令：

service iptables stop

在终端验证

如果大家在输入指令之后出现如下的提示，首先的建议是再输入一次，看看是否还是没有可用的datanodes

若连续两次都是没有可用的datanode，可以尝试输入指令把safemode关掉：

hdfs namenode -safemode leave

关闭之后，再执行上面的report指令，如果仍然没有可用节点，这时可以先把yarn、dfs关掉，然后再开启一次，一般情况下都可以完成，若仍然不行，找google，囧

打开eclipse

选择好工作目录:

选择完毕之后，进入eclipse的页面：

看主界面，不晓得大家是否有一种亲切感

新建项目，测试上传下载

建好java项目之后，就是导入jar包，我们可以从hadoop中的share文件目录中找，先右键项目，然后选择properties选项

在弹出的窗口中选择java build path

按图中顺序选择：

在弹出的新窗口中，选择图中红色框选项

在弹出的新窗口中，选择图中红色框选项

输入jar包名称

按图中顺序选择选项：

首先选择hdfs的主要jar包

添加完毕之后，要再点一次依赖包，然后再点添加jar包，如果不点，它没有选中，就会导致后面的添加不在这个包内。

选择hdfs的以来包，注意路径，在lib文件夹中，要选择里面的所有jar包。

添加完毕之后，再次点击依赖包，然后再添加其他jar包

接下来添加common的主要jar包：

添加完毕之后，记得点击依赖包再添加其他jar包！！！

添加common的依赖jar包，也在lib目录中，注意路径!!!

到此相关的jar包添加完毕，点击完成：

点击完成之后在退回来的窗口中，点击完成：

在退回来的窗口中继续点击完成：

添加完成后，目录中的显示：

eclipse快捷键设置

虽说linux环境下的eclipse跟windows环境下外观一样，可是有个常用的快捷键与windows不一样，所以我还是按照之前的习惯，把linux中eclipse的提示键也修改成ALT+/

点击window的properties选项

在弹出的对话框中，在左侧的检索框输入keys,然后点击搜索到的标签，出现如下页面：

在内容框中输入content assist ，找到后点击这条搜索结果，在下面红色框的bending栏中，修改成ALT+/

修改完毕后记得保存！

当然同时还要看看有没有跟刚刚设置的快捷键冲突的指令：

可以暂时先把冲突的指令设置成空的，修改完毕后点完成退出窗口

前期工作准备完成，下面开始代码！
