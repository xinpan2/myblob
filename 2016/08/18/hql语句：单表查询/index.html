在hibernate中，有下列比较常用的查询语句：

  • hql （hibernate query language）
  • criteria queries （条件查询）
  • native queries（原生态sql语句查询）

最常用的就是hql,那么 使用hql进行查询，又分为：单表查询，一对多和多对多查询，多表查询。

首先我们来说一下单表查询：

就拿获取所有课程来说，我们用hql语言查询所有的classes。可以有如下方法：

方法一

List<Classes> classes=session.createQuery("from Classes").list();

方法二

List classes=session.createQuery("select cid,cname from Classes").list();

这里的每一个得到的list里面，是数组。而第一种跟第三种list里面的数据为对象。如果一定要带参数查询的话，看第三种方法，比较靠谱

方法三

List classes=session.createQuery("select new cn.ansel.domain.Classes(cid,cname)from Classes").list();

这种方法的目的是：我们想通过带有参数来选择所有的classes，但是又不想得到的结构为（第2种方法）那样的情况。在这一步中，要保证classes有空参构造函数，这种方法得到的classes结构为：

大家可能观察到，这种方法跟第一种 直接(“from Classes”)得到的List集合的结构是一样的，那么为什么还要这种呢？

比如说，当这个表中我们要用到的字段，就只有名字和id,但是这个表中的字段一共有几十个，那么如果还是利用第一种，查询出所有字段的话，效率会降低，加上浪费内存，所以这种情况适合用这一种new的方法。
