在linux中安装软件，有两种，一种是自动安装（rpm格式的安装包），另外一种是手动安装（tar.gz格式的安装包）

rpm安装包

虽然RPM软件打上了红帽系统的表情，但是在很多linux项目都适用。首先来讲一下rpm软件包的安装。

首先准备好rpm格式的软件安装包，在这里我准备了mysql的安装包（如果大家要下载可以在文章结尾处中的参考资料找到下载链接，或者自己去官网下载）。

准备好软件之后，打开之前的secureCRT软件，建立好连接，并且把安装包放在容易用命令行输入的位置，在这里我把安装包放在c盘的根目录，我顺便把等下要用到的手动安装包也放到c盘

现在可以借助secureCRT（以后简称CRT），按ALT+P，打开一个新标签页面:

这是一个ssh模仿ftp的方式上传文件，通过put 文件位置 上传到用户的根目录下:

此时代表上传成功，当然，如果不知道文件的具体名称，可以跟上面的操作一样，使用tab来提示，现在返回到linux查看文件

开始安装：

在命令行中输入如下命令

rpm -ivh MySQL-server-5.1.73-1.glibc23.i386.rpm

  • i：表示install
  • v：过程及结果是visible的
  • h：输出到控制台上的信息更加人性化

提示权限不够，我们使用root来执行

sudo rpm -ivh MySQL-server-5.1.73-1.glibc23.i386.rpm

接下来使用动态图展示:

安装失败了，根据提示信息可以看到:

file /usr/share/mysql/charsets/Index.xml from install of MySQL-server-5.1.73-1.glibc23.i386 conflicts with file from package mysql-libs-5.1.66-2.el6_3.i686

提示说文件有冲突，现在把冲突的依赖包找出来删掉，才能成功安装,这时，看到安装提示，很多名称，但是都跟mysql有关，所以按关键字搜索:

sudo rpm -qa | grep mysql

  • q:query
  • a:all

只找到一个，所以把这个删掉。

sudo rpm -e mysql-libs-5.1.66-2.e16_3.i686

  • e:删除

此时提示还有其他软件依赖这些包，要强行删除才能成功安装mysql，所以在命令中加上如下参数：

sudo rpm -e mysql-libs-5.1.66-2.el6_3.i686 --nodeps

此时再进行安装:

在图中出现了^C的字眼，是刚刚我想弄成动态图，可是已经输入了安装命令，并且不能停止……

注意图中红色框部分内容，是用来设置密码的。

我们选择下面那一种方法,通过指令：

/usr/bin/mysql_secure_installation

需要有一个mysql客户端，所以把客户端也安装了:

客户端安装成功，继续输入刚刚设置密码的命令:

在一开始输入错误2次密码，是因为我没有看清楚提示，它要求的是mysql的当前登录密码，我输入了linux的登入密码。现在尝试登陆mysql

自动安装的好处就是，傻瓜式安装，就像windows中的无脑下一步，还把很多参数帮我们配置好了
