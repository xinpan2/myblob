选择排序与冒泡排序比较

选择排序是选好了再交换，冒泡排序是边判断边交换，虽然选择排序和冒泡排序的平均时间复杂度都是O(n^2),但是交换消耗的CPU比判断多，所以选择排序比冒泡排序效率高。

堆排序

堆排序是对于选择排序的优化，选择排序对于排序n个元素的数组，找到最大/最小值，需要n-1次判断，并且每次都要进行n-1次，其里面肯定有很多步骤是重复的，为了能让元素的判断和比较”更有效率“，J. W. J. Williams在1964年提出了更“一劳永逸”的办法-堆排序，让每次比较所得到的结果，对于下一次都是有用的，而不是像选择排序那样，每次都重新进行n-1次判断。

什么是堆？

在维基百科中是这样定义的：

英文版：

In computer science, a heap is a specialized tree-based data structure that satisfies the heap property: If A is a parent node of B then the key of node A is ordered with respect to the key of node B with the same ordering applying across the heap. A heap can be classified further as either a “max heap” or a “min heap”. In a max heap, the keys of parent nodes are always greater than or equal to those of the children and the highest key is in the root node. In a min heap, the keys of parent nodes are less than or equal to those of the children and the lowest key is in the root node. Heaps are crucial in several efficient graph algorithms such as Dijkstra’s algorithm, and in the sorting algorithm heapsort. A common implementation of a heap is the binary heap, in which the tree is a complete binary tree (see figure).

译文:

在计算机科学领域中，堆是一种特别的基于树形的数据结构，它满足堆的性质：若节点A是节点B的双亲节点，那么节点A对应的值与节点B对应的值的顺序关系与堆中其他节点关系一样。堆分为“大根堆”、“小根堆”，在大根堆中，根节点的值总是大于或等于它所有子节点的值；在“小根堆”中，根节点的值总是小于或等于其所有子节点对应的值。堆对于某些高效图形算法如戴克斯特拉算法和对于排序算法中的堆排序的实现非常重要，堆的共同实现是二叉堆，二叉堆里面的树是完全二叉树。

中文版:

堆（英语：Heap）是计算机科学中一类特殊的数据结构的统称。堆通常是一个可以被看做一棵树的数组对象。在队列中，调度程序反复提取队列中第一个作业并运行，因为实际情况中某些时间较短的任务将等待很长时间才能结束，或者某些不短小，但具有重要性的作业，同样应当具有优先权。堆即为解决此类问题设计的一种数据结构。

从堆的定义中我们知道堆的性质为:

若以层次遍历的顺序按1开始编号，k代表该堆的数组实例，则节点之间满足下面关系

  • 大根堆：根节点为i,左子节点为2i,右子节点为2i+1

    • k[i]>=k[2i] 且k[i]>=k[2i+1]

  • 小根堆：节点编号与大根堆一样，只是节点间的关系相反

    • k[i]<=k[2i] 且 k[i]<=k[2i+1]

思想

  • 在这里我们用大根堆来实现堆排序

  • 首先把待排序的序列构成一个大根堆，在构建完后，大根堆的根节点即为最大值，这时把最大值与待排序数组最后一个元素交换位置。

  • 这样就得到了最大值，然后再从[0,n-1]个元素中，再次构建大根堆，这样得到的跟节点就是第二大的值，再与倒数第二个元素交换位置。

  • 重复上述步骤，知道只有1跟节点为止。

  • 总而言之：就是通过不断的构建大根堆，然后把根节点从倒数第一个元素开始交换，每次交换完记得把长度缩减1，这样就得到了升序序列的数组
