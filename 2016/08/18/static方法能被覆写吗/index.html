反编译的代码中，可以看到，在源代码中调用的testClassMethod()中，反编译的代码却是Animal.testClassMethod

我google了一下，发现并没有我需要的答案，很多人给出了答案，却没有标明出处，于是我把java的官方文档翻了一下，试图找出问题的答案。

If a subclass defines a static method with the same signature as a static method in the superclass, then the method in the subclass hides the one in the superclass.

大意:若子类中定义了一个与父类签名相同的静态方法，那么子类中的静态方法将隐藏父类中的静态方法。

这里我就困惑了，既然隐藏了，那么为什么还调用的是父类的静态方法？而不是子类的静态方法？

官方给出的解释:

The distinction between hiding a static method and overriding an instance method has important implications:

The version of the overridden instance method that gets invoked is the one in the subclass.

The version of the hidden static method that gets invoked depends on whether it is invoked from the superclass or the subclass.

大意：

隐藏一个静态方法与覆写一个实例方法之间的区别有重要的意义：

  • 如果通过父类调用被子类覆写过的实例方法，那么调用的是子类中覆写的方法

  • 对于子父类相同的静态方法，调用与哪一个，取决于哪个类调用。

那么之前源码中通过父类 标识符 =new 子类,然后通过animal.testClassMethod() 执行时，调用的是父类的静态方法，如果是子类调用，那么这里执行的将是子类的静态方法。只不过，父子类中签名相同的静态方法叫做隐藏，这种隐藏最终调用哪个类的方法，取决于哪个类调用，父子类中相同签名的实例方法叫做覆写。同时，在IDE的提示中可以看到（如下图所示），通过实例化对象的方式来调用静态方法是多余的，因为这违背了静态的初衷：直接通过类名调用。
