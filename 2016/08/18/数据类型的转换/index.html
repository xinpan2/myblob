int 2 byte

在java中，我们接触到了八种基本数据类型，byte char short int float long double boolean

在这里，对于数字类型byte short int float long double 作进一步说明。

在java中，给数字赋值默认使用的是整型常量（int），如果把整型常量赋值给byte,char,short,long，整型常量表示的范围没有超过它们的最大范围,就可以进行默认的类型转换，如果超过了相对应的类型，则会报错,要求强制类型转换

比如

byte number1 = 100;

因为byte用一个字节表示，有2^8种状态256种,又因为java使用有符号数来表示数字, 所以byte的取值范围在-128~127，100没有超出byte的表示范围，所以100（整型常量）自动转换成byte而不会损失精度

又比如

byte byte2 = 128;

128不在byte所能表示的范围内，所以写完这句代码时，编译器会报错。

如图所示，编译器要求自行对128进行强制类型转换成byte类型，如果这样做的话，会产生精度损失，即把128的二进制表示截断成8位二进制，把多出来的表示位省去了

int 2 char

char对字符使用unicode编码，char用两个字节表示，又因为编码没有负数，所以char的取值范围为2^16=65536 ，即0~65535 .

所以，出现下面的赋值并不会报错：

char c1 = 65535;

但是出现下面的赋值，就会报错了

int 2 long

当赋值给long类型变量又没有添加Long标识时，若没有超过int的表示范围，则不会报错，若超过了int的表示范围，依然报错

避免这种情况的方法是在整数常量int赋值给long类型变量时，在后面添加一个L，表示long类型。建议使用大写的L，因为小写的l看起来跟数字1很相似，让人以为是数字。修改后就没有报错了
