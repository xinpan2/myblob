在cloudera上搭建Hadoop集群

发表于 2016-10-16   |   分类于 hadoop笔记   |  

最近在进一步学习Hadoop生态体系时，一直听到有人说cloudera搭建集群比较方便，于是我现在也来试试，看看是否真如大家所说的那么方便。

今天稍微看了一下杭州的云栖大会，我就看了一下VIVE和大数据的部分，还有阿里云CEO及马云的演讲，好像喝了鸡汤一样，但是有时间还是要把云栖大会上关于大数据的视频弄到手再看一遍，肯定受益匪浅！只是现在我知识面不够广，所以看起来有些困难~~~加油！现在是DT时代？？！下面进入主题，在cloudera上搭建Hadoop集群。关于搭建CDH，我翻了Google前面20页的文章，但是都没有搭建成功，这个搭建是符合我自己机器的，并且亲测成功之后才发出来的！

安装的centos要64位！！都在说cloudera的维护很好，我在其中注册了一个账号来下载，填了手机号，早上注册的，第二天早上就打电话问我有什么问题，当时是一位女士跟我说她是cloudera公司的，然后她问我有没有什么问题，我当时没听明白，然后她问我你知道cloudera公司吗？我说不知道。。。到后面她又解释一次我才明白，囧~

首先在这里我用到的系统都是新的系统，我用的是刚刚安装完毕的虚拟机，9个节点！设置的ip为192.168.31.11~192.168.31.19 网关还是以前的192.168.31.2 子网掩码是默认的24或255.255.255.0。DNS解析地址需要设置成8.8.8.8，还有网络适配器中的VMNet3其中的DNS也需要设置成8.8.8.8,备用的设置成8.8.4.4,对于设置虚拟机IP的方法我就不在这里说了。

设置完毕之后，设置好高级网络配置中的VMNet8，把网关设置成跟虚拟机一致：

ip地址只要设置成跟虚拟机中的ip地址不冲突即可！

通过修改配置文件/etc/sudoers，通过root用户打开这个文件，通过shift+g移动到文件末端，看到如下所示，红色框内是添加后的内容：

大家可以根据自己的用户名修改之后保存退出，即可生效。

关闭防火墙

在root用户下通过指令：

service iptables stop

上述指令只是短暂关闭，我们还要关闭全局：

chkconfig iptables off

其他节点也要进行这个步骤！

然后重启，以后的防火墙都是关闭状态！

关闭selinux

大家可以暂时把它理解为一个安全系统，通过指令：(root用户下)

sudo vi /etc/sysconfig/selinux

把SELInux=设置成disabled后，保存退出.

其他节点也需要关闭SELInux，设置为关闭之后，重启机器！重启之后，通过如下指令查看SELInux是否关闭：

getenforce

其他节点也需要检查！

设置主机名

大家可以直接通过在命令行中输入hostname就可以看到当前用户：

如果用户名是localhost之类的，可以直接通过

hostname 新的用户名

来设置，不过这种修改方式重启就失效，如果需要长期有效，就需要通过修改/etc/sysconfig/network文件中的hostname字段，使用root用户打开这个文件：

sudo vi /etc/sysconfig/network

其他节点也要执行相同的步骤！

根据自己的需求，修改后保存退出即可。

重启机器

通过在root用户下reboot命令，使机器重启，让刚刚的修改生效。

查看是否有遗留的jdk

重启完毕后，通过命令

rpm -qa | grep java

rpm -qa | grep jdk

来查看是否有遗留的安装包，如果有的话通过命令：

rpm -e 安装包名字

来删除指定的安装包，这种情况我们在前面通过rpm安装包安装MySQL时遇到过。

添加新的用户

因为在安装完Linux之后，都是使用root登录，这时需要创建一个用户来搭建集群，但是考虑到我们在安装Linux时已经新建了一个hadoop用户，所以这一步可以免，但是没有创建用户的同学，需要创建一个用户,通过指令

adduser 用户名

通过

passwd 用户名

来设置指定用户的密码

设置完毕之后，可以通过指令

su 用户名

来连接到指定的用户，连接完毕之后，可以通过

whoami

来查看当前连接的用户：

配置本地yum源

如果不配置yum源，我们在安装时访问的就是国外网站，网速会非常慢，甚至安装失败，所以我们安装我们已经下载好的安装包。

httpd

首先我们需要验证httpd是否安装，通过指令

rpm -qa |grep httpd

如果没有显示任何的安装包，说明该Linux没有安装，我们就需要通过指令：

yum install -y httpd

来安装httpd。

启动httpd

首先我们通过指令

service httpd status

来查看httpd是否启动

既然没有启动，我们通过指令：

service httpd start

来启动httpd：

切换root用户再次执行：

图中的提示不能识别server名称，而使用192.168.31.11代替，可以通过：

sudo vi /etc/httpd/conf/httpd.conf

在最后面加上一句：

ServerName xinpan1:80

保存退出即可。然后通过指令

sudo service httpd stop

关闭httpd：

再次启动：

启动完毕之后，在浏览器的地址栏中输入：

xinpan1

或者

xinpan1:80

来访问该节点对应的服务器：

设置httpd开机启动

我们通过

service httpd start

的方式来启动httpd 就像启动防火墙一样，是暂时的，只要重启，httpd又会关闭，所以我们要设置其开机自动启动，首先我们先来看看httpd现在是否自动启动，通过指令：

chkconfig --list|grep httpd

我们像要修改其中的参数，就需要切换到root 用户来修改，因为才会chkconfig是系统命令：

安装lrzsz插件来上传文件

我们先安装lrzsz插件来上传我们刚刚下载的两个文件，通过命令：

sudo yum install -y lrzsz

执行之后，报错，因为镜像问题：

方法一

直接在安装时设置好DNS地址wei8.8.8.8即可！

方法二

首先我们先ping任意一个网站，比如

ping 114.114.114.114

如果能ping通，则说明可以上网，然后通过修改文件：

sudo vi /etc/resolv.conf

在其中添加完DNS解析，保存后退出：

重启网卡,通过指令：

sudo service network restart

重启完毕之后，再次执行安装指令：

sudo yun install -y lrzsz

还是失败，于是我们直接跳转到root用户，然后直接执行：

echo "nameserver 8.8.8.8" >> /etc/resolv.conf

后，再运行安装lrzsz插件:

安装完毕之后，输入rz来上传文件，显示如下窗口：

修改文件拥有者

上传完毕之后，需要修改softwares文件的拥有者，目前的拥有者是root：

所以我们要修改成hadoop（当前用户名），通过指令:

sudo chown -R hadoop:hadoop softwares/

解压文件

把刚上传的gz文件，解压到服务器中，通过指令：

tar -zxvf cm5.4.1-centos6.tar.gz -C /var/www/html/

权限不足，通过root用户解压：

sudo tar -zxvf cm5.4.1-centos6.tar.gz -C /var/www/html/

通过服务器访问

在浏览器地址栏中输入：

xinpan1/cm

创建地址

我们需要把访问的链接修改成跟官网一样：

而我们的只是domain/cm，所以把cm5/redhat/6/x86_64/拷贝，然后在html目录中创建这个目录，通过指令：

sudo mkdir -p cm5/redhat/6/x86_64/

  • -p为递归创建

创建完毕之后，把我们解压的cm目录移动到刚刚创建的目录中：

sudo mv cm cm5/redhat/6/x86_64/

通过浏览器访问，把刚刚redhat的访问地址复制到新标签，然后把主机名改成我们的节点名进行访问：

http://archive.cloudera.com/cm5/redhat/6/x86_64/

修改后：

http://xinpan1/cm5/redhat/6/x86_64/

创建yum源

我们需要为刚刚访问的地址配置yum源，这样在安装时就会安装我们下载好的安装包，而不是去网站下载，首先我们进入/etc/yum.repos.d/：

通过命令

sudo vi cloudera-manager.repo

创建文件，文件名大家也可以自定义，输入如下内容：

  • [cloudera-manager]：名称
  • name：要与上面中括号内容一致，否则会报错
  • baseurl：即我们上传到服务器的安装包，进入到5这个目录是因为我们安装的rpm包在这个目录中

  • gpgcheck:gpg检查，0即关闭

在浏览器中访问刚刚设置的baseurl：

这样yum源配置完毕。
